# Step 1 - Text-based metadata enrichment

Step 1 enriches the Krovets folk paintings and icons subset with:

- structured candidate tags tags generated from record metadata (title - description)

This stage processes 312 records and produces machine-generated candidate tags using NLP techniques, thus paving the way for **Step 2**, where tags are extracted by captions generated through image recognition.

## Overview

The pipeline uses a two-stage approach:

1. **Downloading stuctured metadata of the Folk Paintings subset**  
   `notebooks/01_metadata_extraction_from_europeana.ipynb`
2. **Candidate tag extraction from metadata**  
   `notebooks/02_tag_extraction_from_metadata.ipynb`

The second stage utilises NLP techniques such as **rule-based heuristics** and **spaCy NER**

## Inputs and Outputs

### Stage 1

- **Inputs**: Europeana Set API ([set/26106](https://api.europeana.eu/set/26106.json))
- **Outputs**: `krovets_folk_metadata.csv` with one row per record and normalized metadata fields.

### Stage 2

- **Inputs**: `krovets_folk_metadata.csv` - structured metadata for the folk paintings and icons subset of the Krovets collection, generated by `01_metadata_extraction_from_europeana.ipynb`.
- **Outputs**: `krovets_folk_tags.csv` with columns `europeana_id`, `Figures`, `Objects`, `Scenes`.

## Usage

You can run this workflow in **Google Colab** or in a **local Jupyter environment**.

### 1) Prepare environment

- Python 3.10+ recommended
- Install notebook dependencies used in the notebooks (pandas, requests, tqdm, spacy
and other related packages)

### 2) Download records from Europeana API

Run:

- `notebooks/01_metadata_extraction_from_europeana.ipynb`

This notebook:

- fetches collection items' ids through pagination from Europeana Set API: `https://api.europeana.eu/set/26106.json`
- fetches full metadata for each item in the collection through batches using the Europeana Search API:
`https://api.europeana.eu/record/v2/search.json`
- parses each record into a flat row, containing all necessary columns (some may require additional handling or preprocessing)


API authentication notes:

- Create a Europeana account and generate an API key
- In Colab, save the key in Secrets as `EUROPEANA_API_KEY`
- Requests use API key in headers as `X-API-KEY` (not query parameter)

Confirm downloaded images are available locally under `data/folk_images/`
(or your mapped drive folder if running in Colab).

Expected output:
- `outputs/krovets_folk_metadata.csv`

### 3) Generate candidate tags

Run:

- `notebooks/02_tag_extraction_from_metadata.ipynb`

Expected output:

- `outputs/krovets_folk_tags.csv`

## Limitations

- Tags are **AI-generated suggestions**, not final annotations.
- Tags are often general ("man", "woman") and may miss specific named entities
(for example, a depicted person such as "Taras Shevchenko").
- Some extracted tags like scenes are full phrases/sentences (for example, "The Kazan icon of the Mother of God") rather than concise normalized terms.
